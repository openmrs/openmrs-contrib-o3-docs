# Prerequisite knowledge

Before jumping into building with O3, you should be familiar with HTML, CSS, and JavaScript. If you're totally new to frontend development, it's especially important that you develop a good grasp of the fundamentals. Below is a list of the technologies we use in O3 and some recommended resources to learn them.

## HTML & CSS

A solid understanding of HTML and CSS fundamentals is essential for frontend development. Here are some recommended resources:

- [MDN's HTML basics](https://developer.mozilla.org/en-US/docs/Learn/Getting_started_with_the_web/HTML_basics)
- [MDN's CSS basics](https://developer.mozilla.org/en-US/docs/Learn/Getting_started_with_the_web/CSS_basics)
- [CSS Tricks - A Complete Guide to Flexbox](https://css-tricks.com/snippets/css/a-guide-to-flexbox/)
- [CSS Grid Garden](https://cssgridgarden.com/) - An interactive game to learn CSS grid
- [Josh Comeau's CSS course](https://www.joshwcomeau.com/css/interactive-guide-to-flexbox/)
- [web.dev's Learn CSS](https://web.dev/learn/css/)

## JavaScript

Knowing JavaScript is a prerequisite to everything else. We recommend going through the following resources:

- [Introduction to JavaScript - Codecademy](https://www.codecademy.com/learn/introduction-to-javascript)
- [JavaScript to know for React - Kent C. Dodds](https://kentcdodds.com/blog/javascript-to-know-for-react)
- [The Modern JavaScript Tutorial](https://javascript.info/)
- [JavaScript for impatient programmers](https://exploringjs.com/impatient-js/toc.html)
- [Josh Comeau's Operator lookup](https://www.joshwcomeau.com/operator-lookup/)
- [Visual guide to references in JavaScript](https://daveceddia.com/javascript-references/)

## React

The [official docs](https://react.dev/) are a great way to get acquainted with React. **Don't gloss over [the fundamentals](https://react.dev/learn/thinking-in-react)**! Having a solid understanding of its core concepts is very important in helping you learn how to develop high-quality frontend modules in O3. We also recommend going through the following resources:

- [Common mistakes with React - Josh Comeau](https://www.joshwcomeau.com/react/common-beginner-mistakes/)
- [Understanding useMemo and useCallback](https://www.joshwcomeau.com/react/usememo-and-usecallback/)
- [Writing Resilient Components - Dan Abramov (dated but still very useful)](https://overreacted.io/writing-resilient-components/)
- [A (Mostly) Complete Guide to React Rendering Behavior - Mark Erikson](https://blog.isquaredsoftware.com/2020/05/blogged-answers-a-mostly-complete-guide-to-react-rendering-behavior/)
- [Why React Re-renders - Josh Comeau](https://www.joshwcomeau.com/react/why-react-re-renders/)
- [Beginners Guide to React - Kent C. Dodds (dated but still very useful)](https://egghead.io/courses/the-beginner-s-guide-to-react)
- [A Visual Guide to React Rendering](https://alexsidorenko.com/blog/react-render-cheat-sheet/)
- [React Re-renders Guide - Nadia Makarevich](https://www.developerway.com/posts/react-re-renders-guide)
- [When Does React Render Your Component - Zhenghao hao](https://www.zhenghao.io/posts/react-rerender)

- [TypeScript essentials](https://www.totaltypescript.com/books/total-typescript-essentials)
- [TypeScript in 5 minutes](https://www.typescriptlang.org/docs/handbook/typescript-in-5-minutes.html)
- [Total TypeScript](https://www.totaltypescript.com/tutorials)
- [React + TypeScript Cheatsheets](https://github.com/typescript-cheatsheets/react)

## JavaScript tooling

- [Yarn](https://yarnpkg.com/) and [Corepack](https://nodejs.org/api/corepack.html) - many OpenMRS repos use Yarn as the package manager and pin the exact Yarn version in the `packageManager` field of `package.json`. Run `corepack enable` once so Node can activate the pinned Yarn version for each repo.
- [turbo](https://turbo.build/repo) - we use `turbo` to run scripts during local development and our continuous integration processes.
- [swc](https://swc.rs/) - we use SWC to compile our JavaScript and TypeScript code into a format all major browsers support.
- [Rspack](https://rspack.dev/) - most current O3 frontend modules use Rspack to bundle code for local development and production.
- [webpack](https://webpack.js.org) - O3's build tooling still supports Webpack-compatible configuration, and a few legacy modules still use Webpack directly. You will mostly encounter Webpack concepts through Module Federation, loader configuration, and older frontend modules.

## Git

It is imperative that you keep your code in a version control system. OpenMRS uses [Git](https://git-scm.com/). You should know the basics of using Git and GitHub. Great learning resources include:

- Learn [Git Branching](https://learngitbranching.js.org/)
- [Git and GitHub for Beginners](https://www.youtube.com/watch?v=RGOj5yH7evk)
- [Dangit, Git](https://dangitgit.com/en)
- [OpenMRS Guide to using Git](https://wiki.openmrs.org/display/docs/Using+Git)

## OpenMRS APIs and data model

OpenMRS provides APIs the frontend uses to request data. We recommend getting familiar with these APIs and the OpenMRS
data model. Find the relevant links below:

- [OpenMRS data model](https://devmanual.openmrs.org/technology/datamodel/)
- [REST API](https://rest.openmrs.org/)
- [FHIR API](https://om.rs/fhir)
